# HomeUp 泛家居营销SaaS平台
演示地址：youkailong:5082
## 项目简介

HomeUp 是面向泛家居行业的全链路 SaaS 增长平台，包含总部运营端、门店管理端、导购微信小程序三端一体。以375px 移动端自适应为基础，统一登录与角色权限，实现产品展示、客户管理、营销获客、业绩数据、物料采购、培训赋能的数字化闭环，帮助品牌总部标准化赋能终端，提升门店获客转化与经营效率。

## 项目特性

### 三端一体
- **总部运营端**：全局数据管理、品牌运营、供应链管理、培训赋能
- **门店管理端**：门店运营、客户管理、订单管理、员工管理
- **导购微信小程序**：客户跟进、产品展示、营销获客、业绩查看

### 核心功能
- **客户管理**：客户信息管理、标签体系、跟进记录、客户分析
- **营销获客**：转介绍裂变、AI智能话术、表单工具、活动管理
- **销售管理**：订单管理、电子合同、在线收款、业绩统计
- **服务管理**：安装工单、售后工单、量房管理、服务评价
- **培训赋能**：课程学习、考试认证、技能提升
- **数据分析**：销售数据、客户数据、门店数据、营销数据

### 技术特性
- **响应式设计**：一套代码适配多端，移动端优先
- **模块化架构**：组件化开发，提高代码复用性
- **现代化技术栈**：Vue 3 + Vite + Pinia + Element Plus + Vant
- **性能优化**：代码分割、图片懒加载、缓存策略
- **无障碍适配**：符合WCAG 2.1 AA级标准

## 安装与运行

### 环境要求
- Node.js >= 16.0.0
- npm >= 7.0.0

### 安装依赖
```bash

```

## 核心功能模块

### 导购端 (guide)
- **工作台**：数据概览、快捷功能、待办事项
- **客户管理**：客户列表、客户详情、跟进记录
- **AI智能**：话术建议、客户分析、跟进提醒
- **营销工具**：转介绍、表单工具、分享功能
- **服务管理**：工单管理、量房管理、售后跟进
- **产品管理**：产品列表、产品详情
- **个人中心**：个人信息、业绩统计、培训学习

### 门店端 (store)
- **工作台**：门店数据、销售趋势、待办事项
- **客户管理**：客户分配、行为分析、跟进管理
- **订单管理**：订单列表、订单详情、生产进度
- **服务管理**：工单处理、安装调度、质量检查
- **营销管理**：活动管理、短信营销、数据统计
- **采购管理**：物料采购、供应商管理、订单管理
- **财务管理**：发票管理、应收账款、对账管理

### 总部端 (headquarters)
- **数据中心**：销售数据、门店数据、客户数据
- **品牌运营**：活动管理、社区运营、短信营销
- **供应链管理**：生产管理、物流管理、库存管理
- **门店管理**：门店列表、门店详情、运营分析
- **财务管理**：返利管理、对账管理、发票管理
- **培训赋能**：课程管理、考试管理、培训活动
- **系统设置**：用户管理、角色管理、菜单管理

## 设计规范

项目遵循严格的设计规范，确保视觉一致性和用户体验。设计规范包括：

- **色彩系统**：品牌主色 #667EEA，辅助色和中性色体系
- **字体系统**：八级标准化字号，Web端使用Inter/思源黑体，移动端使用苹方/思源黑体
- **间距系统**：8px基准栅格，所有间距、内边距、宽高、定位必须为8px的整数倍
- **组件规范**：按钮、输入框、卡片、导航栏等组件的设计标准
- **页面布局**：各终端页面的布局规范和视觉元素要求
- **交互规范**：核心交互逻辑、动效参数、反馈机制

详细设计规范请参考 `docs/设计规范.md` 文件。

## 开发流程

### 分支管理
- **main**：主分支，用于发布生产版本
- **develop**：开发分支，用于集成新功能
- **feature/**：功能分支，用于开发新功能
- **bugfix/**：修复分支，用于修复bug

### 代码规范
- 遵循Vue 3 Composition API风格
- 使用ESLint和Prettier进行代码检查和格式化
- 组件命名使用PascalCase，文件命名使用kebab-case
- 变量命名使用camelCase，常量使用UPPER_CASE
- 函数命名使用camelCase，类名使用PascalCase

### 提交规范
- 提交信息格式：`[模块名] 简要描述修改内容`
- 例如：`[guide] 修复工作台数据加载问题`

### 开发步骤
1. 从develop分支创建功能分支
2. 实现功能，遵循设计规范和代码规范
3. 运行测试，确保功能正常
4. 提交代码，创建Pull Request
5. 代码审查通过后，合并到develop分支
6. 发布前，从develop分支合并到main分支
