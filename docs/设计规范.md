# 泛家居营销SAAS平台设计规范

## 一、设计总览

### 1.1 设计风格定位
- **核心风格**：现代简约、专业商务、科技感
- **行业属性**：泛家居行业，注重品质感与专业性
- **目标用户**：导购员、门店管理者、总部运营人员
- **设计理念**：以用户为中心，简化操作流程，突出核心功能，提供清晰的视觉引导

### 1.2 适配范围
- **移动端**：320px-768px，以375px为基准宽度
- **PC端**：769px-1920px，以1920px为基准宽度
- **响应式适配**：一套设计兼容多端，确保在不同设备上都有良好的用户体验

## 二、基础设计系统（原子层）

### 2.1 色彩系统

#### 2.1.1 主色
- **品牌主色**：#667EEA（靛蓝色）
  - HEX: #667EEA
  - RGB: 102, 126, 234
  - HSL: 234°, 66%, 65%
  - CSS变量: --primary-color
  - **使用场景**：主按钮、核心高亮、品牌元素、导航栏

#### 2.1.2 辅助色
- **成功**：#00B42A（绿色）
- **警告**：#FF7D00（橙色）
- **错误**：#F53F3F（红色）
- **信息**：#165DFF（蓝色）

#### 2.1.3 中性色
- **纯白**：#FFFFFF
- **背景**：#F5F7FA
- **边框**：#E4E7ED
- **辅助文本**：#909399
- **正文**：#606266
- **标题**：#303133
- **纯黑**：#000000

#### 2.1.4 色彩使用规则
- 同页面主色占比≤30%
- 辅助色≤10%
- 中性色≥60%
- 所有色彩必须通过WCAG 2.1 AA级对比度校验

### 2.2 字体系统

#### 2.2.1 字体选择
- **Web端**：Inter / 思源黑体
- **移动端**：苹方 / 思源黑体

#### 2.2.2 字号规范（八级标准化）
- 36px：大标题
- 30px：一级标题
- 24px：二级标题
- 20px：三级标题
- 18px：四级标题
- 16px：正文
- 14px：辅助文本
- 12px：标注文本

#### 2.2.3 行高规范
- 标题：1.2-1.3
- 正文：1.5-1.6
- 标注文本：1.4

#### 2.2.4 字重规范
- 400：常规（正文）
- 500：中等（重点标注）
- 600：半粗体（副标题）
- 700：粗体（主标题）

### 2.3 间距与栅格系统

#### 2.3.1 基准单位
- **8px基准栅格**：所有间距、内边距、宽高、定位必须为8px的整数倍

#### 2.3.2 间距梯度
- 8px：元素内边距、小间距
- 16px：元素间距、模块内边距
- 24px：模块间距、页面内边距
- 32px：大模块间距、页面边距
- 48px：超大模块间距
- 64px：页面级间距

#### 2.3.3 栅格系统
- **Web端**：12列栅格，槽宽24px，页面边距32px
- **平板**：8列栅格，槽宽16px，页面边距24px
- **移动端**：4列栅格，槽宽16px，页面边距16px

### 2.4 圆角与阴影规范

#### 2.4.1 圆角规范
- 4px：小按钮、输入框
- 8px：按钮、卡片
- 12px：大卡片、弹窗
- 16px：大弹窗、特殊容器
- 24px：圆形元素、头像

#### 2.4.2 阴影规范
- **一级阴影**：悬浮元素
  - box-shadow: 0 2px 8px rgba(0, 0, 0, 0.08)
- **二级阴影**：卡片、弹窗
  - box-shadow: 0 4px 16px rgba(0, 0, 0, 0.12)
- **三级阴影**：全局遮罩弹窗
  - box-shadow: 0 8px 32px rgba(0, 0, 0, 0.16)

### 2.5 图标规范
- **图标库**：统一使用Vant和Element Plus内置图标
- **尺寸规范**：8px的倍数，如16px、24px、32px
- **风格**：线性图标，线条粗细一致
- **语义**：图标必须与操作含义完全匹配

## 四、页面布局与视觉设计

### 4.1 终端选择页
- **布局**：垂直居中布局，卡片式设计
- **视觉元素**：品牌Logo、终端选项卡片、背景渐变
- **交互**：卡片hover效果，点击反馈

### 4.2 登录页
- **布局**：垂直居中布局，表单式设计
- **视觉元素**：品牌Logo、登录表单、背景渐变
- **交互**：表单验证，登录按钮状态变化，验证码倒计时

### 4.3 导购移动端首页（工作台）
- **布局**：顶部导航栏 + 内容区域 + 底部导航栏
- **视觉元素**：数据概览卡片、快捷功能图标、待办事项列表
- **交互**：卡片点击效果，下拉刷新，加载更多

### 4.4 门店管理端首页
- **布局**：左侧导航栏 + 顶部导航栏 + 内容区域
- **视觉元素**：数据概览卡片、销售趋势图表、最近订单列表
- **交互**：图表交互，表格排序，分页

### 4.5 总部运营端首页
- **布局**：左侧导航栏 + 顶部导航栏 + 内容区域
- **视觉元素**：数据概览卡片、销售趋势图表、门店销售排行
- **交互**：图表交互，表格排序，分页

## 五、交互与动效规范

### 5.1 核心交互逻辑
- **点击反馈**：所有可点击元素必须有视觉反馈
- **表单验证**：实时验证，错误提示清晰
- **导航切换**：平滑过渡，状态明确
- **数据加载**：加载动画，骨架屏

### 5.2 动效参数
- **动画时长**：200ms-300ms
- **缓动函数**：ease-in-out
- **过渡效果**：opacity + transform

### 5.3 反馈机制
- **成功提示**：绿色背景，成功图标
- **错误提示**：红色背景，错误图标
- **警告提示**：橙色背景，警告图标
- **信息提示**：蓝色背景，信息图标

## 六、无障碍与性能优化

### 6.1 无障碍适配
- **对比度**：文本与背景对比度≥4.5:1
- **键盘导航**：支持Tab键导航
- **屏幕阅读器**：支持ARIA属性

### 6.2 性能优化
- **图片懒加载**：使用loading="lazy"
- **代码分割**：按需加载组件
- **缓存策略**：合理使用localStorage
- **动画优化**：使用transform/opacity，避免重排重绘

## 七、设计与代码使用指南

### 7.1 设计文件管理
- 所有设计文件统一存放于`docs/设计/`目录
- 设计文件命名规范：`[页面名称]_设计稿.sketch`

### 7.2 代码实现规范
- 严格遵循设计规范，1:1还原设计
- 使用CSS变量管理全局样式
- 组件化开发，提高代码复用性
- 响应式适配，确保多端兼容

### 7.3 开发流程
1. 参考设计规范确定页面风格
2. 搭建页面结构
3. 实现核心功能
4. 优化交互体验
5. 进行多端适配测试
6. 性能优化与无障碍检查
