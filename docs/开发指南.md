# HomeUp 开发指南

## 一、开发环境搭建

### 1.1 环境要求

- **Node.js**：>= 16.0.0
- **npm**：>= 7.0.0
- **Git**：>= 2.0.0

### 1.2 安装步骤

1. **克隆仓库**
   ```bash
   git clone [仓库地址]
   cd home_up
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动开发服务器**
   ```bash
   npm run dev
   ```

4. **访问项目**
   打开浏览器访问 `http://localhost:5173`

## 二、代码规范

### 2.1 代码风格

- **Vue 组件**：使用 Vue 3 Composition API
- **JavaScript**：遵循 ES6+ 语法
- **CSS**：使用 Sass 预处理器，遵循 BEM 命名规范

### 2.2 命名规范

- **组件命名**：使用 PascalCase，如 `GuideLayout.vue`
- **文件命名**：使用 kebab-case，如 `guide-layout.vue`
- **变量命名**：使用 camelCase，如 `customerName`
- **常量命名**：使用 UPPER_CASE，如 `MAX_COUNT`
- **函数命名**：使用 camelCase，如 `getCustomerList()`
- **类名命名**：使用 PascalCase，如 `CustomerService`

### 2.3 代码组织

- **组件拆分**：按照功能和职责拆分组件
- **逻辑分离**：将业务逻辑与 UI 逻辑分离
- **工具函数**：将通用功能抽离为工具函数

### 2.4 注释规范

- **组件注释**：使用 JSDoc 格式注释组件
- **函数注释**：使用 JSDoc 格式注释函数
- **代码注释**：对复杂逻辑添加注释说明

## 三、开发流程

### 3.1 分支管理

- **main**：主分支，用于发布生产版本
- **develop**：开发分支，用于集成新功能
- **feature/**：功能分支，用于开发新功能
- **bugfix/**：修复分支，用于修复 bug

### 3.2 开发步骤

1. **创建分支**
   ```bash
   git checkout develop
   git pull origin develop
   git checkout -b feature/[功能名称]
   ```

2. **开发功能**
   - 遵循设计规范和代码规范
   - 实现功能需求
   - 编写单元测试

3. **提交代码**
   ```bash
   git add .
   git commit -m '[模块名] 简要描述修改内容'
   git push origin feature/[功能名称]
   ```

4. **创建 Pull Request**
   - 在 GitHub/GitLab 上创建 Pull Request
   - 描述功能实现和修改内容
   - 等待代码审查

5. **合并代码**
   - 代码审查通过后，合并到 develop 分支
   - 删除功能分支

### 3.3 代码审查

- **代码风格**：检查代码是否符合规范
- **功能实现**：检查功能是否按需求实现
- **性能优化**：检查代码是否存在性能问题
- **安全性**：检查代码是否存在安全隐患

## 六、状态管理

### 6.1 Pinia 使用

项目使用 Pinia 进行状态管理，按照功能模块划分不同的 store：

- **user**：用户信息和认证状态
- **finance**：财务相关状态
- **installer**：安装师傅相关状态
- **order**：订单相关状态
- **referral**：转介绍相关状态
- **service**：服务相关状态

### 6.2 状态管理最佳实践

- **单一数据源**：所有状态都存储在 store 中
- ** mutations**：使用 mutations 修改状态
- ** actions**：使用 actions 处理异步操作
- ** getters**：使用 getters 计算派生状态

## 七、路由管理

### 7.1 路由配置

项目使用 Vue Router 进行路由管理，按照终端和功能模块组织路由：

- **/auth**：认证相关路由
- **/guide**：导购端路由
- **/store**：门店端路由
- **/headquarters**：总部端路由

### 7.2 路由守卫

实现路由守卫，确保用户在访问需要认证的页面时已经登录，同时根据用户角色控制页面访问权限。

## 八、API 调用

### 8.1 API 封装

项目使用 Axios 进行 API 调用，封装了统一的请求和响应处理：

- **请求拦截**：添加认证令牌、处理请求参数
- **响应拦截**：处理错误响应、统一错误提示

### 8.2 API 接口分类

按照功能模块分类 API 接口：

- **认证接口**：登录、登出、刷新令牌
- **客户接口**：客户列表、客户详情、客户跟进
- **订单接口**：订单列表、订单详情、订单状态更新
- **服务接口**：工单管理、安装调度、服务评价
- **营销接口**：转介绍、活动管理、短信发送
- **数据接口**：销售数据、客户数据、门店数据

## 九、UI 开发

### 9.1 设计规范

项目遵循严格的设计规范，确保视觉一致性和用户体验：

- **色彩系统**：品牌主色 #667EEA，辅助色和中性色体系
- **字体系统**：八级标准化字号，Web端使用Inter/思源黑体，移动端使用苹方/思源黑体
- **间距系统**：8px基准栅格，所有间距、内边距、宽高、定位必须为8px的整数倍
- **组件规范**：按钮、输入框、卡片、导航栏等组件的设计标准

### 9.2 组件库使用

- **PC端**：使用 Element Plus 组件库
- **移动端**：使用 Vant 组件库
- **自定义组件**：根据设计规范开发自定义组件

## 十、性能优化

### 10.1 代码优化

- **代码分割**：使用动态导入，按需加载组件
- **组件懒加载**：减少初始加载时间
- **Tree Shaking**：移除未使用的代码

### 10.2 资源优化

- **图片懒加载**：使用 loading="lazy" 属性
- **图片压缩**：优化图片大小
- **静态资源缓存**：合理设置缓存策略

### 10.3 渲染优化

- **虚拟列表**：处理大量数据的列表
- **防抖和节流**：优化事件处理
- **减少重排重绘**：使用 transform/opacity 进行动画

## 十四、常见问题

### 14.1 依赖安装失败

- 检查 Node.js 版本是否符合要求
- 检查网络连接是否正常
- 尝试使用 `npm install --force` 强制安装

### 14.2 开发服务器启动失败

- 检查端口是否被占用
- 检查项目配置是否正确
- 尝试删除 `node_modules` 目录并重新安装依赖

### 14.3 构建失败

- 检查代码是否存在语法错误
- 检查依赖是否正确安装
- 检查构建配置是否正确

### 14.4 功能无法正常使用

- 检查 API 调用是否正确
- 检查状态管理是否正确
- 检查路由配置是否正确
- 检查权限设置是否正确
