# HomeUp UI设计规范文档

## 一、设计风格定位

### 1.1 核心风格
- **现代简约**：简洁明了的界面设计，减少视觉干扰
- **专业商务**：符合企业级应用的专业感和可信度
- **科技感**：融入现代科技元素，提升用户体验

### 1.2 行业属性
- **泛家居行业**：注重品质感与专业性，体现家居行业的温馨与舒适

### 1.3 目标用户
- **导购员**：需要简洁易用的界面，快速完成工作
- **门店管理者**：需要数据清晰的界面，便于决策
- **总部运营人员**：需要功能全面的界面，便于管理

### 1.4 设计理念
- **以用户为中心**：关注用户需求，提供直观的操作体验
- **简化操作流程**：减少操作步骤，提高工作效率
- **突出核心功能**：明确功能优先级，突出核心操作
- **提供清晰的视觉引导**：通过色彩、布局等元素引导用户操作

## 二、适配范围

### 2.1 移动端
- **屏幕尺寸**：320px-768px
- **基准宽度**：375px
- **适配策略**：响应式设计，确保在不同尺寸的移动设备上都有良好的用户体验

### 2.2 PC端
- **屏幕尺寸**：769px-1920px
- **基准宽度**：1920px
- **适配策略**：响应式设计，确保在不同尺寸的PC设备上都有良好的用户体验

### 2.3 响应式适配
- **设计原则**：一套设计兼容多端，确保在不同设备上都有良好的用户体验
- **断点设置**：
  - 移动端：< 768px
  - 平板：768px - 1024px
  - PC端：> 1024px

## 三、色彩系统

### 3.1 主色
- **品牌主色**：#667EEA（靛蓝色）
  - HEX: #667EEA
  - RGB: 102, 126, 234
  - HSL: 234°, 66%, 65%
  - CSS变量: --primary-color
  - **使用场景**：主按钮、核心高亮、品牌元素、导航栏

### 3.2 辅助色
- **成功**：#00B42A（绿色）
  - 使用场景：成功提示、状态标识
- **警告**：#FF7D00（橙色）
  - 使用场景：警告提示、状态标识
- **错误**：#F53F3F（红色）
  - 使用场景：错误提示、状态标识
- **信息**：#165DFF（蓝色）
  - 使用场景：信息提示、状态标识

### 3.3 中性色
- **纯白**：#FFFFFF
  - 使用场景：背景、卡片
- **背景**：#F5F7FA
  - 使用场景：页面背景
- **边框**：#E4E7ED
  - 使用场景：边框、分割线
- **辅助文本**：#909399
  - 使用场景：辅助说明、占位符
- **正文**：#606266
  - 使用场景：正文文本
- **标题**：#303133
  - 使用场景：标题、重要信息
- **纯黑**：#000000
  - 使用场景：特殊强调

### 3.4 色彩使用规则
- 同页面主色占比≤30%
- 辅助色≤10%
- 中性色≥60%
- 所有色彩必须通过WCAG 2.1 AA级对比度校验

  - 使用场景：模块标题
- **24px**：二级标题

## 五、间距与栅格系统

### 5.1 基准单位
- **8px基准栅格**：所有间距、内边距、宽高、定位必须为8px的整数倍

### 5.2 间距梯度
- **8px**：元素内边距、小间距
- **16px**：元素间距、模块内边距
- **24px**：模块间距、页面内边距
- **32px**：大模块间距、页面边距
- **48px**：超大模块间距
- **64px**：页面级间距

### 5.3 栅格系统
- **Web端**：12列栅格，槽宽24px，页面边距32px
- **平板**：8列栅格，槽宽16px，页面边距24px
- **移动端**：4列栅格，槽宽16px，页面边距16px

## 六、圆角与阴影规范

### 6.1 圆角规范
- **4px**：小按钮、输入框
- **8px**：按钮、卡片
- **12px**：大卡片、弹窗
- **16px**：大弹窗、特殊容器
- **24px**：圆形元素、头像

### 6.2 阴影规范
- **一级阴影**：悬浮元素
  - box-shadow: 0 2px 8px rgba(0, 0, 0, 0.08)
- **二级阴影**：卡片、弹窗
  - box-shadow: 0 4px 16px rgba(0, 0, 0, 0.12)
- **三级阴影**：全局遮罩弹窗
  - box-shadow: 0 8px 32px rgba(0, 0, 0, 0.16)

## 七、图标规范

### 7.1 图标库
- **统一使用**：Vant和Element Plus内置图标

### 7.2 尺寸规范
- 8px的倍数，如16px、24px、32px

### 7.3 风格
- **线性图标**：线条粗细一致，保持视觉统一

### 7.4 语义
- 图标必须与操作含义完全匹配，确保用户能够直观理解

## 九、页面布局与视觉设计

### 9.1 终端选择页
- **布局**：垂直居中布局，卡片式设计
- **视觉元素**：品牌Logo、终端选项卡片、背景渐变
- **交互**：卡片hover效果，点击反馈

### 9.2 登录页
- **布局**：垂直居中布局，表单式设计
- **视觉元素**：品牌Logo、登录表单、背景渐变
- **交互**：表单验证，登录按钮状态变化，验证码倒计时

### 9.3 导购移动端首页（工作台）
- **布局**：顶部导航栏 + 内容区域 + 底部导航栏
- **视觉元素**：数据概览卡片、快捷功能图标、待办事项列表
- **交互**：卡片点击效果，下拉刷新，加载更多

### 9.4 门店管理端首页
- **布局**：左侧导航栏 + 顶部导航栏 + 内容区域
- **视觉元素**：数据概览卡片、销售趋势图表、最近订单列表
- **交互**：图表交互，表格排序，分页

### 9.5 总部运营端首页
- **布局**：左侧导航栏 + 顶部导航栏 + 内容区域
- **视觉元素**：数据概览卡片、销售趋势图表、门店销售排行
- **交互**：图表交互，表格排序，分页

## 十、交互与动效规范

### 10.1 核心交互逻辑
- **点击反馈**：所有可点击元素必须有视觉反馈
- **表单验证**：实时验证，错误提示清晰
- **导航切换**：平滑过渡，状态明确
- **数据加载**：加载动画，骨架屏

### 10.2 动效参数
- **动画时长**：200ms-300ms
- **缓动函数**：ease-in-out
- **过渡效果**：opacity + transform

### 10.3 反馈机制
- **成功提示**：绿色背景，成功图标
- **错误提示**：红色背景，错误图标
- **警告提示**：橙色背景，警告图标
- **信息提示**：蓝色背景，信息图标

## 十一、异常状态/空状态/加载状态设计

### 11.1 异常状态
- **404页面**：友好的错误提示，返回按钮
- **500页面**：系统错误提示，重试按钮
- **网络异常**：网络连接错误提示，刷新按钮

### 11.2 空状态
- **无数据**：友好的空状态提示，引导用户操作
- **无权限**：权限不足提示，引导用户联系管理员
- **搜索无结果**：搜索无结果提示，建议用户调整搜索条件

### 11.3 加载状态
- **页面加载**：骨架屏
- **数据加载**：加载动画
- **按钮加载**：按钮加载状态

## 十二、设计组件复用说明

### 12.1 组件复用原则
- **优先使用**：项目现有的组件库
- **统一风格**：确保组件风格一致
- **按需定制**：根据业务需求定制组件

### 12.2 组件库使用
- **PC端**：使用Element Plus组件库
- **移动端**：使用Vant组件库
- **自定义组件**：根据设计规范开发自定义组件

## 十三、与现有设计系统的适配方案

### 13.1 设计系统一致性
- **色彩系统**：与现有设计系统保持一致
- **字体系统**：与现有设计系统保持一致
- **组件规范**：与现有设计系统保持一致

### 13.2 适配策略
- **渐进式适配**：逐步将现有组件替换为符合设计规范的组件
- **兼容性**：确保新旧组件的兼容性
- **文档同步**：及时更新设计文档，确保文档与实现一致
